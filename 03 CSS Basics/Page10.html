CSS Info

You can imagine that every piece of content is wrapped in an imaginary box:

BLOCK-LEVEL tags - their content takes up the entire width of the container (body, header, div, etc.)
INLINE-LEVEL tags - don't trty to take up the entire width of their container and flow along with the content/text around them (anchor, span, img, input, label, etc.)

With CSS you can change the layout from "Block --> Inline" -or- "Inline --> Block"

Box Model
There are 4 main components to content:
  1. Content Area - actual content
  2. Padding - layer around edges of content area
  3. Border
  4. Margin - space that can be added outside the border
Summary
  • Each tag's content fits in an inviable box.
  • Each block-level tag's box takes up an entire line.
  • Padding can be used to adjust the spacing within a container.
  • Margin can be used to adjust the spacing betrween containers.

CLASS attribute - can be added to an HTML tag so you can access it directly with CSS

*GOOD RULE OF THUMB*
-First declare the broadest rules with type selectors, and then get more specfic with the class selectors.

More IMAGE Notes:
-Background images are not handled in the HTML, but rather the CSS (as well as background colors)
-Looks like:

body {
  background-image: url(images/img.jpg);
  background-position: *par1 *par2;
  background-repeat: repeat;
}

*par 1 = Verticle alignment (top, center, or bottom)
*par 2 = Horizontal alignment (left, center, or right)

**Background-position: can also be repeat-x, repeat-y, or no-repeat.


Background-image can be shortened and written on one line as follows:
body {
  background: #color url(images/img.jpg) top left repeat;
}
